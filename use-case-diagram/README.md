# Use Case Diagram

This use case diagram shows how different users interact with the Airbnb Clone backend system.

## Overview
The diagram illustrates the main actors — Guest, Host, and Admin — and their key actions:
- **Guest:** Register, search, book, pay, and review properties.  
- **Host:** Add and manage property listings, view bookings.  
- **Admin:** Manage users, listings, and payments.

It provides a clear overview of system interactions and core backend functionalities.
