This document lists the main user stories from the Airbnb Clone use case diagram.

## Guest
1. As a guest, I want to create an account and log in so that I can book properties.  
2. As a guest, I want to search and filter properties so that I can find a suitable place.  
3. As a guest, I want to book a property and make payments so that I can confirm my stay.  
4. As a guest, I want to leave reviews after my stay so that I can share my experience.

## Host
5. As a host, I want to add, edit, and delete my property listings so that I can manage my rentals.  
6. As a host, I want to view all my bookings so that I can track guest reservations.

## Admin
7. As an admin, I want to manage users, properties, and payments so that I can maintain the platform.
